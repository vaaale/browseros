# The desktop, windows, and dock

BOS presents a familiar desktop metaphor: a wallpaper with icons, a top bar, a
dock, and floating windows.

---

## Desktop icons

Built‑in and installed apps appear as icons on the desktop (top‑left, laid out in
a fixed order). **Double‑click** an icon to open the app in a new window.

The desktop refreshes automatically when apps are **installed**, **uninstalled**,
**restored**, or **purged** — you don't need to reload the page for a new app's
icon to show up.

---

## The dock

The dock at the bottom lists apps for quick access. Click an app in the dock to
**launch** it (or **focus** it if it's already open).

---

## The top bar

The top bar runs across the top of the screen. When BOS is served through its
**Supervisor** (live version control), the top bar also shows the **version
controls** — a dropdown of branches plus Promote/Discard actions for previewing
candidate versions of BOS. See
[Live version control](../versions/live-version-control.md). When BOS is run
normally, the version controls are simply hidden.

---

## Windows

Each open app lives in its own window.

- **Move** — drag the title bar.
- **Resize** — drag the bottom‑right corner. Windows have a minimum size so they
  never collapse to nothing.
- **Maximize / restore** — double‑click the title bar, or use the maximize button.
- **Minimize** — hides the window; focusing the app again restores it.
- **Close** — the close button removes the window.
- **Focus** — click anywhere on a window to bring it to the front.

The three coloured buttons on a window are **close**, **minimize**, and
**maximize**.

### Singleton apps

Some apps allow only **one window at a time**: **Assistant**, **Memory**,
**Docs**, and **Settings**. Launching one of these again just focuses (and
un‑minimizes) the existing window rather than opening a duplicate. **Files** and
**Browser** can have several windows open at once.

---

## Text selection

You can **select and copy text** anywhere inside app content — chat messages,
file editors, and documentation. Only the desktop "chrome" (icons, the dock, the
top bar, and window title bars) opts out of selection so dragging windows feels
natural.

---

## Wallpaper

Change the wallpaper from **Settings → Appearance** (a preset gradient, an image
URL, or an image from your files), or directly from the **Files** app: open an
image and choose **Set as wallpaper**. See
[Appearance](../settings/appearance.md).
